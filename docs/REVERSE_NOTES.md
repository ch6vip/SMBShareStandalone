## 逆向流程

### 1. Stub 壳分析

```java
// com.stub.StubApp (stub classes.dex)
// 360加固壳 Application类
// 入口: libjiagu.so → interface5/interface7/interface8 等 native 方法
// 真实 Application: "com.example.ourom.MyApplication" (base64解码)
```

`StubApp` 通过 `libjiagu.so` 加载真实的 Application 类。stub 中包含大量 `n0*` 和 `n1*` 混淆的 native 方法，实际业务逻辑在 `libourom.so` 中。

### 2. 字符串解密

原始 app 使用 XOR 16 加密敏感字符串:

```java
// com.tianyu.util.a
public static String a(String str) {
    char[] charArray = str.toCharArray();
    for (int i = 0; i < charArray.length; i++) {
        charArray[i] = (char) (charArray[i] ^ 16);
    }
    return String.valueOf(charArray);
}
```

### 3. SMB 字符串提取

所有 SMB 相关字符串以 **Base64** 编码存储在 `libourom.so` 中。

#### 提取步骤

```powershell
# 1. 从 libourom.so 提取所有 base64 候选字符串
$content = [System.IO.File]::ReadAllBytes("libourom.so")
$text = [System.Text.Encoding]::UTF8.GetString($content)
$matches = [regex]::Matches($text, "[A-Za-z0-9\+/=]{40,}")

# 2. 解码并过滤 SMB 相关
foreach($m in $matches) {
    try {
        $decoded = [System.Text.Encoding]::UTF8.GetString(
            [System.Convert]::FromBase64String($m.Value)
        )
        if($decoded -match 'smb|gvfs|smbd|nmbd|samba') {
            Write-Output $decoded
        }
    } catch {}
}
```

### 6. 未解之谜

以下信息因依赖动态加载和远程配置，无法完全提取:

1. **123云盘 OAuth Token 获取逻辑** — token/client_id/secret 值通过 `libourom.so` 中的加密字符串返回，需要 IDA Pro 动态调试才能提取实际值。当前使用的 public URL 如果失效，需要更新。

2. **gvfsd.sh 脚本内容** — 该脚本在 123云盘上 (需要认证)，无法获取。但其作用是启动 gvfsd，核心逻辑与 start_smb.sh 相同。

3. **smb3.tgz 内部结构** — 包含 Samba 4.x 编译的 arm64 二进制文件:
   - `smbd0` — SMB 服务端守护进程
   - `dbus-daemon` — D-Bus 消息总线
   - `lib/` 目录 — Samba 运行时依赖库
   
   这些二进制文件是现成编译好的 SambaDroid 产物，非自行开发。

4. **设备管理功能** — 原始 app 的 SMB 功能可能是为了在局域网内多设备之间分发 ROM/系统镜像。`/mnt/rtempi/system_rom/system1-5/` 目录结构表明这是多系统 ROM 管理系统的一部分。
